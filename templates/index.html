ML Systems

4 Models. 4 Insights.

Each model is a trained Random Forest Classifier, independently specialized for a single prediction task.

🎯
MODEL 01
KT Risk Predictor

Predicts whether a student will face a Keep Term (KT) next semester based on their Semester 1 performance. Uses 6 features including attendance, marks averages, and existing KT count.

Random Forest Binary Classification 6 Features
Output: KT Likely / No KT + probability %
⚠️
MODEL 02
Dropout Risk Analyzer

Classifies each student into High, Medium, or Low dropout risk using CGPA, total KTs, attendance, average grade score, live KTs, and KT attempt level.

Random Forest 3-Class 6 Features
Output: High / Medium / Low Risk + probabilities
💼
MODEL 03
Placement Predictor

Estimates campus placement chances using CGPA, projects, internships, communication skills, professor reviews, attendance, and KT count. 7-feature model.

Random Forest 3-Class 7 Features
Output: High / Moderate / Low placement chance
📚
MODEL 04
Weak Subject Detector

Predicts performance grade (A/B/C/D) for each subject using UT marks, semester marks, internal marks, practical marks, and attendance. Identifies the weakest subject per student.

Random Forest Grade A–D 5 Features
Output: Grade A / B / C / D + weakest subject flagged
System Pipeline

How It Works

ASPRS uses a modular ML pipeline where each model specializes in one task. Student data flows through preprocessing into the prediction engine, which runs all models and aggregates results into a clear dashboard.

A separate StandardScaler is paired with each model, ensuring inputs are normalized using the exact same parameters applied during training.

Upload Excel → instant batch predictions for all students
All 4 models run independently via REST API
350–400 decision trees per model for accuracy
Download results as formatted Excel with one click
Per-student breakdown + class-wide summary stats
1

Data Upload

Faculty uploads an Excel sheet with student records — marks, attendance, CGPA, skills, etc.

2

Feature Mapping

Input columns are mapped to the exact feature vectors each model was trained on.

3

Normalization

Each model's StandardScaler transforms the data to match training distribution.

4

Random Forest Inference

Hundreds of decision trees vote on the output. Class probabilities computed from vote ratios.

5

Results Dashboard

Per-student predictions shown on screen with summary stats and Excel download option.
